# 成品目标

构建《西游记 · 三借芭蕉扇》：一段约 45-90 分钟、桌面浏览器可玩的单人**回合制指令 RPG 完整战役**，
《梦幻西游》《问道》那样一眼就认出的回合制西游——指令出招、速度行动顺序、五行相克、
法术与法力、携宠协战、阵型、七十二变——跟着原著第五十九至六十一回从火焰山脚走到
牛魔王两阶段决战与还扇西行。

第一版的质量标志是：中文玩家一眼认出的回合制西游玩法、忠于原著的战役、明代木刻套印
的辨识度，以及每回合有真决策（技能/相克/变化/携宠/真扇 明显强于无脑普攻）。

首发界面只实现简体中文。所有玩家可见文案集中管理，不把文字烙进图片；字体覆盖人物、
地名和法宝字形并允许商业发布。保留语言资源边界，为后续本地化留出替换能力。

# 必读设计

按顺序读取：

1. `analysis/SOURCE_BIBLE.md`
2. `concepts/CONCEPT.md`
3. `design/GAME_DESIGN.md`
4. `design/ART_DIRECTION.md`

`source/西游记.txt` 只用于回看第五十九至六十一回的事实，不要把完整原文塞进实现上下文，
也不要用现代影视记忆覆盖策划。

# 必须保真

## 战斗契约

- 指令回合制：每回合先为每位伙伴下指令（攻击/法术/防御/道具/特技/自动/逃跑），
  下完后双方按**速度从高到低**依次行动、逐个结算、飘出伤害数字。
- 五行相克成体系：金→木→土→水→火→金，克制 ×1.5（飘「克!」）、被克 ×0.66。
- 命中、暴击、伤害浮动为概率，但**全程走可设种子的 RNG**，保证自测同种子同操作可复现。
- 法术耗法力、物理不耗；防御减伤并回法力；携宠（辟水金睛兽，水克火）作为第四参战单位。
- 悟空「七十二变」临时改五行/属性/技能；阵型（天罡/六丁）改攻防速；都要有实际战术价值。
- 成长系统：升级发潜力点自由加点（体/攻/防/速/灵）、法术随等级熟练进阶、剧情掉落简单武器装备、
  规则型法宝（定风丹/避火锦，不加攻）、召唤兽剧情门控捕捉与上阵、物品背包；不做抽卡、门派数值墙、
  装备打造与交易、PVP 和 MMO 长线。

## 原作因果

- 火焰山八百里阻断西路，需借翠云山芭蕉洞铁扇仙（罗刹女，牛魔王之妻）的芭蕉扇。
- 罗刹女因红孩儿旧怨拒借；战斗教学中悟空变化「化虫入腹」可触发直接取胜（第59回）。
- 先得假扇，假扇使火更旺、火中生出火兵；辟水金睛兽水克火（第60回素材）。
- 真扇在牛魔王处；悟空骗扇、牛魔王化八戒反骗，最终积雷山决战（第61回）。
- 牛魔王现白牛真身（多阶段）；真扇三段一息火、二生风、三落雨，还扇后师徒西行。

## 视觉风格

- 明代彩印神怪 / 木刻套印 / 明代绣像版画质感；忠于公版原著与明代版画、戏台、彩塑。
- 全屏木刻战斗背景（火焰山/翠云山/积雷山/序幕）+ 敌左上我右下经典对阵；下方留承载 UI 与单位。
- 角色立绘保留戏曲亮相轮廓与五行辨识；牛魔王人形↔白牛真身体量升级；宠辟水金睛兽。
- 卷轴木框界面（宣纸米/墨/朱/金线/暗木）：指令台、行动顺序条、HP/MP、飘字（暴击更红更大、
  金色「克!」、治疗绿字、色盲用文字冗余）。真扇三段以世界重印（息火/生风/落雨）呈现。

# 范围

必须包含：

- 序幕·火焰山脚：可点击行走的小世界、土地问话、进入战斗入口；
- 战斗1·罗刹女：教学（指令/速度序/相克/暴击）+ 化虫入腹彩蛋；
- 战斗2·火焰山火兵：携宠水克火、假扇反噬；
- 战斗3·牛魔王：两阶段（人形→白牛真身）+ 真扇三段翻盘 + 还扇西行结局；
- 标题/开场引导、五行相克可读性、阵型/七十二变/携宠、升级、存读、失败重试。

明确排除：抽卡、门派数值、PVP、MMO 长线养成、经济系统、自由大地图、可杀死并改写原著
存活角色的分支、多人和后端。

# 实现自由

根据环境自行选择框架、渲染、数据结构、状态管理、输入映射、原创程序化资产和测试入口。
优先做完灰盒战斗核心与可复现结算，再补木刻立绘、背景与反馈；不要为了展示技术静默改成
实时动作或卡牌构筑。数据表驱动：新增角色/敌人/技能/阵型/战斗只加数据不改引擎。

范围压力出现时，依次压缩过渡演出、额外战斗和养成深度；不得删除指令回合、速度结算、
五行相克、携宠、变化、多阶段 Boss、真扇三段和 seedRNG 可复现。

# 完成证据

运行：`cd build/app && python3 -m http.server 5173`，浏览器打开 `http://127.0.0.1:5173/`
（可选 `?seed=42&fast=1` 用于可复现自测）。

- `node test/battle.mjs`：纯引擎自测，固定种子逐字节可复现，覆盖五行系数、伤害/暴击序列、
  速度行动顺序、阵型修正、三场战斗胜负、化虫入腹彩蛋、法力消耗与防御回蓝、假扇反噬、
  真扇三段、Boss 阶段转换、逃跑规则、升级解锁。
- `python3 test/qa_browser.py`：playwright 走通 标题→序幕→三场战斗→结局→读档，
  断言控制台 0 报错并保存过程截图。
- 手动验证：战役可通关且 Boss 有挑战不劝退；智取路线（技能/相克/变化/携宠/真扇）明显优于无脑普攻。

本示例已交付可运行游戏（`build/app/`）与本构建说明。运行命令、验证结果与已知限制见
`build/app/RUN.md`。
