# 来源说明

仓库收录 Project Gutenberg eBook #52200《金瓶梅》，传统署名兰陵笑笑生。上游是崇祯本
（绣像本）系统的百回足本，繁体正文。

## 版本判定

第一回作「西門慶熱結十弟兄　武二郎冷遇親哥嫂」，而非词话本的「景陽岡武松打虎　潘金蓮
嫌夫賣風月」，据此判定为崇祯本系统。第一百回作「韓愛姐路遇二搗鬼　普靜師幻度孝哥兒」。

## 完整性

正文自第一回至第一百回连续完整，共 100 回，回目行格式为「第X回　上聯　下聯」。

## 洁本处理（重要）

仓库内 `金瓶梅.txt` 是**删节本**，不是上游足本。上游足本含大量露骨性描写，本示例既不
需要这部分内容，也不适合随仓库公开分发。

删节由 `expurgate.py` 从上游原文可复现地生成：

```bash
python3 expurgate.py <上游 pg52200.txt> 金瓶梅.txt
```

- 以句为单位，命中过滤词表的句子整句删除，连续删除合并为一处 `〔此處刪節〕` 标记，
  使删节可见而非静默抹去。
- 该版本的情色段落成块出现，因此**一个段落若命中两处以上，首末命中之间的整段一并删除**，
  包括中间未命中的连接句。
- 紧邻删节处的「詞曰／有詩為證」等韵文框架一并删除；情色场面惯用韵文起收。
- 回目（章回标题）不删：属于书目数据，其中数条本身含敏感词（第61回「燒陰戶」、
  第73回「白綾帶」），且 `scripts/validate_repo.py` 依赖其完整。

**两份互相独立的词表**（这是第一版失败后加上的关键设计）：

- `FILTER_LEXICON` 负责删除，同时覆盖**文言雅称**（麈柄／龜頭／雲雨）与**白话直陈**
  （肏／奶頭／酥胸／淫水／精泄／澡牝）两个语域。
- `AUDIT_LEXICON` **不参与删除**，只用于判定退出码。

第一版只列了文言雅称、漏掉了本书最常用的白话层，导致 43 回共 87 句露骨描写残留；
而当时的自检用**同一份词表**过滤又验收，属于循环论证，必然报告「无残留」。现在验收
由一份没有参与过滤的词表完成，`expurgate.py` 在检出任何残留时以非零码退出并打印实例。

实际结果：触及 287 个段落、删除 1,484 句、保留 723,836 字、留下 287 处删节标记，
涉及 85 回。删节较密的是第4、13、52、59、62、72、78、82 回。

## 完整性与残留检查

```bash
python3 expurgate.py <上游 pg52200.txt> 金瓶梅.txt   # 退出码 0 才算通过
```

当前状态：100 回完整；独立词表检出残留 **0** 句。

需要说明的是，基于词表的删节是**尽力而为，不是形式化完备**——它能保证所列词汇不再
出现，不能证明不存在任何用完全未列词汇写成的段落。因此本文件不作绝对承诺；若发现
残留，请扩充 `FILTER_LEXICON` 并重新生成，同时把新词补进 `AUDIT_LEXICON` 之外的
第三方检查。

## 授权边界

Project Gutenberg 将该电子书标记为在美国属公有领域，具体许可见上游文件声明。公开
再分发时仍应遵守上游声明和使用地适用法律；在部分司法辖区，发行未删节本另有限制，
这也是本仓库只收录删节本的原因之一。

本示例只依据古典原著重新设计，不使用影视、动画、漫画、游戏或现代译本中新增的造型、
对白与设定；这些现代衍生作品各自可能仍受版权或商标保护。

## 内容边界

本示例的改编方向是明代市井家宅的权力、经济与人情结构，不改编也不暗示情色内容。
所有策划产物、界面文案和美术资产均按此边界执行，具体规则见 `design/ART_DIRECTION.md`
与 `design/GAME_DESIGN.md`。
