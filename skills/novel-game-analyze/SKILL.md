# 小说游戏化拆解

建立游戏设计所需的事实层，不评价文笔，也不在此阶段选择游戏类型。

读取 [gameability-protocol.md](references/gameability-protocol.md)。

原文可以是任意语言。引用保留原文；策划语言与原文不同时附必要译文，并建立角色名、
地名、物件和规则的统一术语表。提取会影响玩家理解的礼制、宗教、神话、幽默、禁忌和
叙事惯例，不因翻译方便把它们替换成另一文化的近似类型标签。

## 输入

优先复用现有写作工程或拆文产物，只回原文补关键证据；只有原始小说时才按章节
抽取后聚合。不要为了统一格式重做已有分析。

## 提取

只保留会影响玩家动作、世界响应或视觉身份的内容：

1. 世界硬规则、代价及互相冲突的原作事实。
2. 可重复动作、资源和状态变化。
3. 地点、通路、门槛、危险和地标。
4. 角色与势力的目标、默认行动和可改变关系。
5. 条件事件、标志性物件、场面、能力和声音。
6. 不可改的原作事实、授权风险和原作未定义的设计空间。

为硬规则、关键角色目标、玩家身份依据和标志性锚点记录章节或文件位置。普通
候选项不必机械附带置信度对象；推断必须明确标成推断或开放设计空间。

## 输出

生成一份 `analysis/SOURCE_BIBLE.md`，包含来源范围、世界身份、原作事实与证据、玩家
身份候选、可玩动词、世界模型、空间与角色能动性、文化语境、术语表、视觉锚点、风险和设计空白。
短表格优先于重复的文档、结构化数据和证据索引。

来源范围只记录会影响复核的版本、覆盖范围和授权边界。下载日期、字节数、校验值和
本地路径只有在复现契约确实需要时才写，不把获取日志当策划内容。

完成前确认：核心规则可追溯；能回答玩家可能做什么、在哪里做、世界如何回应；
没有把一次性剧情冒充核心循环，也没有提前选定游戏类型。
