# 小说游戏化拆解

建立游戏设计所需的事实层，不评价文笔，也不在此阶段选择游戏类型。

读取 [gameability-protocol.md](references/gameability-protocol.md)。

原文可以是任意语言。引用保留原文；策划语言与原文不同时附必要译文，并建立角色名、
地名、物件和规则的统一术语表。提取会影响玩家理解的礼制、宗教、神话、幽默、禁忌和
叙事惯例，不因翻译方便把它们替换成另一文化的近似类型标签。

## 输入

优先复用现有写作工程或**全量**拆文产物，只回原文补关键证据；只有原始小说时才执行
本 skill 的全书管道。已有分析若只覆盖黄金三章或快速预览，只能帮助判断开篇，不能
证明理解了全书。除非用户明确只改编某一卷或片段，否则来源未覆盖全文时不得进入概念设计。

先确认版本、授权边界、目录结构和实际章/卷/场景数。对已有拆文检查摘要数量是否等于
原文单元数、失败记录是否已补齐、聚合结论能否回指原文；不因目录看起来完整就直接信任。

## 全书管道

1. **建立边界**：一次识别全部章、卷或场景，记录编号、标题、起止位置和总数；后续阶段
   使用同一份边界，不重复切分。
2. **逐单元提取**：每章只记录会改变动作、状态、关系、空间、规则、节奏或视听身份的
   事实；长篇按稳定批次处理，并在 `_progress.md` 记录成功、失败和待重试范围。
3. **覆盖门禁**：成功单元数必须等于来源单元数。缺章、空章、重复边界和失败批次先修复，
   不带着未知缺口做全书结论。
4. **分层聚合**：先把相邻章节归成剧情阶段、地点或规则簇，再从全部中层结果归纳世界
   规则、行动者、循环、空间网络、强弱节奏和标志性锚点，避免一次压缩整本书。
5. **回查原文**：硬规则、数值、能力边界、关系变化、关键结局和选定改编段落重新回到
   原文核对；冲突事实并列记录，不用常识补空。
6. **压缩交付**：工作卡只服务覆盖、恢复和复核，最终结论压缩进 `SOURCE_BIBLE.md`；完成后
   删除无复核价值的合并临时文件，不把过程日志包装成策划成果。

可以在运行时并行处理互不重叠的章节批次，但每批必须使用同一提取契约，聚合和事实回查
保持串行。环境不支持并行时逐批串行完成，不能改成抽样。

## 提取维度

只保留会影响玩家动作、世界响应或视觉身份的内容：

1. 世界硬规则、代价及互相冲突的原作事实。
2. 可重复动作、资源和状态变化。
3. 地点、通路、门槛、危险和地标。
4. 角色与势力的目标、默认行动和可改变关系。
5. 条件事件、标志性物件、场面、能力和声音。
6. 不可改的原作事实、授权风险和原作未定义的设计空间。

逐章卡不做剧情复述，使用“发生了什么 → 状态怎样变化 → 对可玩性有什么意义”的最短
记录。为硬规则、关键角色目标、玩家身份依据、转折结局和标志性锚点记录章节或文件位置。
普通候选项不必机械附带置信度对象；推断必须明确标成推断或开放设计空间。

## 输出

生成一份 `analysis/SOURCE_BIBLE.md`，包含来源范围、**全书覆盖**、世界身份、原作事实与
证据、玩家身份候选、可玩动词、世界模型、空间与角色能动性、剧情/情绪节奏、文化语境、
术语表、视觉锚点、风险和设计空白。全书覆盖按连续范围列出每一阶段的关键变化与游戏化
意义，使覆盖集合与来源单元集合完全一致。短表格优先于重复文档、结构化数据和证据索引。

来源范围只记录会影响复核的版本、覆盖范围和授权边界。下载日期、字节数、校验值和
本地路径只有在复现契约确实需要时才写，不把获取日志当策划内容。

完成前确认：覆盖数与来源数一致且无未处理失败；核心规则可追溯；能回答玩家可能做什么、
在哪里做、世界如何回应；理解开篇、中段、结局和跨段回收；没有把一次性剧情冒充核心循环，
也没有提前选定游戏类型。
