# 游戏化全书拆解方法

长篇分析采用稳定切章、逐章原子提取、分层聚合、计数校验、断点恢复和原文回溯。这里
不评价黄金三章、爽点密度或文风，也不生成写作模仿素材，只提炼游戏设计事实。

## 一、先证明读完

### 来源清点

1. 记录作品版本、卷/章/场景分隔方式、总单元数、缺失部分和授权边界。
2. 一次生成稳定边界表：`编号 | 标题 | 起止位置 | 状态`。恢复、并行和回查都使用它。
3. 已有结构化拆文产物优先读取概要、逐章摘要、角色、剧情、设定、节奏和失败记录；
   只有逐章摘要覆盖数与原文边界数一致时，才视为全量拆解。
4. 用户明确只改编一段时，把来源范围标为限定改编，不得把片段结论扩写成全书规律。

### 逐章游戏化卡

每个单元用紧凑记录回答下列问题；没有新信息时写“无新增”，仍计入覆盖：

| 字段 | 只记录什么 |
|---|---|
| 客观变化 | 谁做了什么，哪项状态、关系、资源、位置或认知因此变化 |
| 动作候选 | 可被玩家重复执行且允许选择、反馈或熟练度差异的动作 |
| 规则证据 | 触发条件、代价、例外、反制和可观察后果 |
| 行动者 | 目标、默认行动、资源、底线及本章后的变化 |
| 空间 | 入口、出口、门槛、地标、风险、可改变部分和尺度 |
| 物件与能力 | 输入、效果、限制、归属和是否只出现一次 |
| 节奏与感受 | 压力怎样建立、释放或反转，玩家需通过何种行动感受到 |
| 视听锚点 | 有原文依据且能服务识别或反馈的轮廓、材质、声音和场面 |
| 证据与边界 | 原文位置；推断、冲突、未定义和现代衍生混入风险 |

逐章卡不是章节摘要。若删除“游戏化意义”后只剩剧情复述，就继续压缩。

### 客观白描

「客观变化」和「规则证据」只写可观察事实：把「谁做了什么」立即转成「哪项状态、关系、
空间或硬规则因此改变」，评价句、总结句一律改写成能被玩家观察或能写进系统的状态变化。

- 「林风展现了强大实力」→「林风三招击败金丹修士 → 练气期可越两阶战斗（能力边界证据）」
- 「得知后山是禁地」→「长老宣布剑冢禁止练气期进入，违者废修为 → 空间门槛＋规则代价」
- 「二人关系破裂」→「张启当众夺走陈默储物袋 → 两人由同门转为仇敌（关系翻转）」

### 批次与恢复

- 短篇可逐章处理；长篇按能完整容纳原文和输出的连续范围分批，不跨章截断因果。
- 每批完成即更新 `_progress.md`：范围、成功数、失败数、重试状态和下一范围。
- 并行批次互不重叠；合并时按原编号排序，检查缺号、重号、空输出和边界漂移。
- 临时合并文件过大时按卷或剧情阶段继续分层聚合，不为了“一次读入”丢弃中段。
- 并行章节批以只读游戏化事实提取为任务；通俗题材的暴力、复仇、情爱、黑暗伦理是常规
  叙事元素，照常提取世界规则与动作事实，不得以“敏感”拒绝整章而造成缺章、破坏覆盖门禁。
- 续跑只接受含边界表的 `_progress.md`；边界表缺失或结构不完整时先从“建立边界”重建再续跑，
  避免不同阶段用不同切分真值。

覆盖门禁只有一个：

```text
成功单元集合 == 来源边界集合
```

不使用“覆盖率大致足够”。无法读取的部分必须在来源范围和风险中明确列出，此时产物只能
标记为部分分析。

## 二、从事实得到系统

把叙事证据转成问题，而不是自动映射：

| 叙事证据 | 游戏问题 |
|---|---|
| 反复动作 | 哪些动作允许选择、反馈和熟练度差异？ |
| 世界规则 | 哪些输入、代价和后果能形成系统？ |
| 关系与势力 | 玩家行动如何改变其行为？ |
| 地点与事件 | 哪些条件可以触发、延迟或改写它们？ |
| 情绪承诺 | 玩家通过什么行动而不是对白感受到它？ |
| 标志性意象 | 哪些元素无需说明就能在截图中认出原作？ |
| 文化语境 | 哪些礼制、神话、幽默、禁忌或价值关系不能被近似翻译替代？ |

机制候选至少需要玩家决策、可读的世界反馈和持续到足以影响下一步的状态。最好还
有多种结果或执行质量，并能通过重复产生学习。一次性高潮可以保留为标志性场面，
但不能假装成循环。

原作事实只用四种边界：`immutable`、`adaptable`、`open`、`conflicted`。不要默认主角
必须是玩家；同时检查主角、原创角色、势力/聚落和观察者等身份能访问哪些动作与
决策空间。

## 三、分层聚合

不要让模型从数百章卡片一步跳到结论。按三层压缩：

1. **单元层**：逐章游戏化卡，保留局部事实与变化。
2. **阶段层**：按剧情目标、地点、能力阶段或规则簇聚合，写清开始状态、压力升级、
   转折、结果和可重复结构。
3. **全书层**：比较所有阶段，找出反复成立的世界因果、角色能动性、空间语法、情绪
   承诺和独有视听身份。

全书层还要主动找反例：某项能力何时失效、规则何时被绕过、角色目标何时改变、结局
怎样回收早期设定。只在开篇成立的规律不能写成世界硬规则。

## 四、事实保真

- 数值、等级、距离、口令、能力边界、亲属/势力关系、死亡与最终归宿必须回查原文。
- 聚合结论引用最早建立、典型运作、重要反例和最终回收位置，不用几十条重复引文证明。
- 同一事实出现冲突时保留各自位置并标为 `conflicted`；版本差异与角色说谎分开处理。
- 原作未定义的内容写入开放设计空间，不能因为某个机制“应该如此”就冒充原作事实。
- 缺失写“原文未明确”，禁编造填空：原文没给的硬事实（数值上限、境界高低、能力边界、
  势力/亲属关系）写“原文未明确”，不用近义项、别处范例或题材惯例补一个看似合理的值。
- 跨指代不串：相邻实体的属性绝不互相挪用——坐骑/法宝战力≠使用者战力、A 的台词或底线
  ≠B 的、本章的数量或口令≠相邻章；合并角色能动性与势力/亲属关系时最易串，逐条认准归属再写。
- SOURCE_BIBLE 的硬规则、数值、能力边界和关系应能 grep 回原文；命中不到的改写为
  “原文未明确”或删除，不保留无源断言。

## 五、交付压缩

`SOURCE_BIBLE.md` 不是拆书数据库。它只保留会影响后续概念选择的事实，并包含一张全书
覆盖表：连续范围、核心变化、反例/回收和游戏化意义。覆盖表的章回集合必须等于来源边界
集合。逐章卡与临时批次可在分析完成后归档或删除，但 `_progress.md` 要保留覆盖和失败结论。
