# 可游戏化判断方法

把叙事证据转成问题，而不是自动映射：

| 叙事证据 | 游戏问题 |
|---|---|
| 反复动作 | 哪些动作允许选择、反馈和熟练度差异？ |
| 世界规则 | 哪些输入、代价和后果能形成系统？ |
| 关系与势力 | 玩家行动如何改变其行为？ |
| 地点与事件 | 哪些条件可以触发、延迟或改写它们？ |
| 情绪承诺 | 玩家通过什么行动而不是对白感受到它？ |
| 标志性意象 | 哪些元素无需说明就能在截图中认出原作？ |
| 文化语境 | 哪些礼制、神话、幽默、禁忌或价值关系不能被近似翻译替代？ |

机制候选至少需要玩家决策、可读的世界反馈和持续到足以影响下一步的状态。最好还
有多种结果或执行质量，并能通过重复产生学习。一次性高潮可以保留为标志性场面，
但不能假装成循环。

原作事实只用四种边界：`immutable`、`adaptable`、`open`、`conflicted`。不要默认主角
必须是玩家；同时检查主角、原创角色、势力/聚落和观察者等身份能访问哪些动作与
决策空间。
