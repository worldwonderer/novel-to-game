# 质量验证契约

## 必须证明

- 页面能够构建、启动并持续渲染，无关键错误或资源失败；
- 真实输入导致玩家或世界产生可观察变化；
- 核心循环能够推进；
- 设计要求的成功、失败或其他结果可到达；
- 重开返回有效初始状态；
- 每个目标视口保持可读且可操作。
- 策划要求的每种界面语言均无缺字、截断、错序或关键术语漂移。

系统规则的验证用状态和结果断言；内容理解与可读性用无提示试玩和真实画面。测试文件
中出现某句策划文案，只能证明文字没变，不能证明规则正确或游戏可玩。

对画布游戏至少比较加载后与输入/时间推进后的真实帧，排除空白、零尺寸、遮挡和只有
背景动画的假运行。截图检查玩家、目标、威胁、界面、反馈和签名画面是否符合
`ART_DIRECTION.md`。

## 严重度

- `blocker`：无法构建/启动、主画面空白、核心输入无效、崩溃或无法完成/重开。
- `major`：关键机制、状态、画面可读性或目标视口失败，但仍有替代路径。
- `minor`：不阻断体验的局部视觉、反馈或警告问题。

保留足以复现结论的命令、状态、日志和截图，不要求为了数量保存重复证据。
