# 对标基准参考：市场 × 题材 × 平台 × 画风

> 用途：需求 intake 阶段，给用户提**有信息量的推荐默认值**（对标名作 / 平台 / 画风 / 单局结构）。所有条目都是**挑选时的锚点**，最终由用户拍板。
>
> **这张表凭通用知识整理，不是已核实的定论。** 真要把某几款拿去当默认或写进交接，先用 WebSearch 联网核实其机制、平台与现况（较新或小众作品尤其容易记错或过时）；联网不可用时标注「凭记忆·未核实」。

## 0. 硬规则：小说语言 → 对标市场

**先按原著语言锁定对标市场，再在该市场里找玩家真正熟悉的爆款。**

| 原著语言 | 对标市场 | 找谁 |
|---|---|---|
| 中文 | 中国大陆 | 国内爆款（原神/一念逍遥/率土/羊了个羊…） |
| 英文 | 西方（欧美） | Steam/主机叙事向（BG3/Disco Elysium/Hades…） |
| 日文 | 日本二次元 | galgame/收集 gacha/JRPG（Fate/蔚蓝档案/P5…） |
| 韩文 | 韩国 | webtoon 改编/即时通讯乙女/工业化放置 RPG（Mystic Messenger/第七史诗…） |

- **对标是为了借具体的玩法结构 / 画风 / 商业原则，不是抄名字。** 每次对标必须落到「借它哪一条」，名字只是索引。
- **目标市场 ≠ 研发地**：不少「最像日系」的爆款其实是韩厂做的（蔚蓝档案=Nexon，NIKKE=Shift Up）。做日韩题材时，题材面向谁（日本宅）与工业打法借谁（韩系）要分开看。

## 3. 画风光谱速查（成本 / 辨识度 / 适配平台）

| 画风 | 一句话 | 代表作 |
|---|---|---|
| **像素** | 成本低–中（高质量像素动画其实不便宜），辨识度中（易同质，靠色彩/题材突围），全平台尤网页/小程序/独立PC | Dead Cells、星露谷 |
| **2D手绘** | 成本中（逐帧动画贵），辨识度高（风格化强），网页/App/PC | 空洞骑士、Gris、茶杯头 |
| **立绘+背景**(galgame式) | 成本低–中（单立绘+CG无需动作），辨识度靠角色设计，全平台尤文字冒险 | Fate、橙光系 |
| **2.5D**(2D伪3D/正交/横版景深) | 成本中，辨识度中–高，网页/App/PC/主机通吃 | 极乐迪斯科、奥日、明日方舟 |
| **卡通渲染3D**(二次元PBR) | 成本高，辨识度高（二次元标配），App/PC/主机，小程序吃力 | 原神、崩坏3 |
| **写实3D** | 成本很高（建模/动捕/光照），辨识度靠制作规格，PC/主机/高端App，**仅史诗/准3A体量才用** | 黑神话悟空、艾尔登法环 |
| **水墨工笔** | 成本中–高（需专门美术），辨识度极高（强文化标签），PC/主机/App 展示向 | 绘真·妙笔千山、墨魂、大神 |

## 4. intake 收尾提醒

- 这些表是**给用户挑选时的锚点与默认值来源**，不是替用户拍板。先按语言定市场 → 按目标平台锁画风与单局结构 → 在题材簇里取 2–4 个对标。
- **对标必须落到「借它哪一条」**：借的是某条明确的玩法结构 / 画风取向 / 商业原则，不是名字装饰。给默认值时把这条原则说出来（例：「一念逍遥式=小程序放置+长线数值买量」而非只报「一念逍遥」）。
- 缺强标杆的题材（系统流、无限流、webtoon复仇/回归、异世界轻改）：诚实标注不确定，给「放置买量 / roguelite / 养成逆袭」等可落地路径**二选一与用户确认**，别承诺原创机制。
