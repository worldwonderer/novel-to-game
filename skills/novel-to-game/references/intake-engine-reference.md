# 引擎选型参考：intake 阶段的生产引擎默认值锚点

> 用途：intake 给用户提"有信息量的引擎默认值"时的速查底本。合成自四块联网研究（网页/H5、客户端/移动、小程序/小游戏、品类专用+原型层），共 **34 次 WebSearch**。**这是起点参考，不是结论**——落地前须复核（见文末）。

## 一、硬规则：引擎受什么约束 & 两层引擎必须分清

**约束轴**：引擎不是自由选的，由 **平台 × 画风 × 题材 × 市场** 共同夹逼。同一部小说，投主机大作、投国内小游戏、投出海 Web，答案完全不同——先定四轴，再点引擎。

**两层引擎（最容易混，务必讲清）**：

| 层 | 是什么 | 怎么选 | 本流水线关系 |
|---|---|---|---|
| **(a) 生产引擎** | 真做成品会用的引擎 | 按平台/画风/题材/市场，从下方主表点名 | intake 给"生产引擎默认值" |
| **(b) 原型层** | 本流水线的可玩交付切片 | **恒为"零构建网页可玩垂直切片"**（当前范例＝原生 ES Module + DOM/Canvas，浏览器直开） | 与选哪款生产引擎**解耦** |

**关键纪律**：原型切片默认 **不用** 生产引擎；选了 Unity/Unreal/Cocos 当生产引擎，也不代表原型要用它。intake 话术＝先按四轴点一个生产引擎默认值，再补一句"原型仍是零构建网页切片，不受该引擎绑定"。

## 三、原型层速查：零构建网页切片怎么实现

**默认永远是手写 web 切片**（零构建、可控体积、可截图迭代）。选择只在下表内做：

| 实现 | 层 | 何时用 |
|---|---|---|
| **Vanilla Canvas/DOM** | 原型默认 | 交付默认；文字/卡牌/像素/极简/UI 重叙事；首屏最小、无构建 |
| **Phaser 4** | 原型（稍重） | 玩法动词＝动作/2D 多物体；生态最厚、文档最全 |
| **PixiJS v8** | 原型（纯渲染器） | 高粒子/华丽 2D/特效/视觉小说；**无内置玩法系统**，玩法自搭或配 Phaser |
| **Three.js** | 原型（3D 轻） | 3D 探索/可视化；灵活但相机/碰撞等引擎层自己搭 |
| **Babylon.js 9** | 原型（3D 重） | 需物理/材质/编辑器的完整 3D 又要留在 Web |
| **生产引擎 Web 导出** | 可选，**非义务** | 仅当切片要验证的正是"成品观感"（见判据） |

**判据一句话**：
- 切片要证明的是 **"玩法"** → 手写 web（vanilla / Phaser / Three），与生产引擎解耦。
- 切片要证明的是 **"成品叙事观感"**（立绘调度/存读档/回放/密集分支）→ 才用能导出 Web 的品类引擎，**优先原生 HTML 系**（Twine 导 HTML、inkjs 挂脚本、Ren'Py/Tyrano 出 web build）。
- **明确禁用**：为了原型去强上 **Unity(Naninovel) / RPG Maker MZ** 的 Web 导出——产物几十 MB、非零构建、迭代慢，与"零构建垂直切片"冲突；这两者只写进"生产层建议"，原型仍用 vanilla/Phaser/Three 重做核心循环。
- Godot/Unity/Cocos/PlayCanvas/Babylon 均可导 Web build 做"更接近成品"的原型，但要接受包体、首屏、移动 web 的代价——**别把"原型必须用生产引擎"当默认**。3D 原型天然用浏览器内 Three.js，正好为将来上 LayaAir/Unity 的 3D 小游戏埋技术路线。

## 四、小程序 / 小游戏专项提醒

**渠道引擎**：Cocos Creator 3.8（几乎全渠道，默认首选）、LayaAir 3.3（真 3D，国内 3D 小游戏市占最高）、Unity+团结引擎（重度 3D 复用，包体/启动成本最重）；**白鹭 Egret 约 2021 已停运，新项目不推荐**；渠道原生框架（微信 `wx.*` / 抖音 `tt.*` + Canvas）适合超轻 2D；Three.js+weapp-adapter 可把网页 3D 搬进微信，但 adapter 模拟不全需手工打补丁。

**包体 / 分包（已核实）**：微信主包 / 单个分包 ≤ **4MB**；开通虚拟支付整包 ≤ **30MB**，否则 ≤ **20MB**；抖音整包 ≤ **20MB**；超出资源走网络下载。**3D 素材极吃包体**，选 LayaAir/Unity 时分包与远程包是必修课。

**发布合规**：国内小游戏**必须备案**（软著 + 适龄提示 + 内容介绍）；开通虚拟支付/收费须**版号**，纯广告变现的免费游戏可暂免版号但仍需备案。

**成人内容禁令**：**成人 / 色情 / 露骨性暗示内容在国内所有小程序小游戏渠道一律禁止上线**（凭国内监管常识，本次搜索未取回渠道条款原文，已标注）。**若源小说属成人向，国内小游戏渠道此路不通**——只能改走出海 App/Web，或做合规化改编。这条直接呼应"NSFW 不能上小程序"。

## 五、起点参考，非结论：落地前务必复核

本文所有版本号、许可条款、平台能力都是**某一时点的联网快照 + 部分凭记忆判断**，会过期。**真要拿去当 intake 默认值或写进交接文档前**，先对拟采用的那一款做 WebSearch 复核：
- 引擎**当前版本**（是否已出新 LTS/大版本）；
- **许可与商业条款**（版税门槛、订阅价、Runtime Fee 类政策变动）；
- **平台能力**（WebGPU 转正与否、Web 导出限制、主机导出路径、小游戏渠道包体/合规规则）。

**联网不可用时**：不得把本文当权威，须在输出里显式标注 **"凭记忆·未核实"**，让下游知道这是待验证锚点而非定论。
