# 美术方向设计方法

核心视觉原则必须能否决一个方案，例如“路线只由世界中的光揭示，不使用漂浮路标”。
“高级、电影感、沉浸、精致”等词不能指导选择。

## 视觉语言

- **镜头与构图**：距离、角度、焦点顺序、前中后景和允许遮挡。
- **形状与尺度**：玩家、目标、威胁、奖励和地标首先靠轮廓区分。
- **世界密度**：细节随决策和节奏变化，不用随机填充制造丰富感。
- **材质、光和颜色**：每个主要选择都说明功能，不依赖单一色相或辉光。
- **运动与声音**：输入、接触、后果和重大状态使用不同强度的反馈。
- **界面**：只显示当前决策需要的信息，并覆盖进行、暂停、失败和结果状态。
- **语言与文化**：字体、字形覆盖、阅读顺序、符号和文本密度服务目标语言；文化元素
  必须有原作依据或明确功能，不把异文化刻板印象当视觉捷径。

## 招牌画面

每帧都是实际游戏画面，不是概念海报。说明当前玩家动作、即时目标与压力、可见的原作
规则、第一视觉焦点、空间层次、功能性色光、运动和界面。缩略图尺寸下仍应看出玩家
在做什么，以及为什么只有这部作品能产生这张画面。

优先借鉴跨媒介的构图、建筑、材质、摄影和声音。高价值角色或物件说明轮廓、材质、
运动和原创边界即可，具体资产来源与制作方式由实现环境决定。

## 生成资产的方向约束

用模型批量产出资产时，方向层面固定三点，制作细节仍交给实现环境：

- **一个风格锚点贯穿整批**：同一套木刻/套色/笔法/年代描述用于每一张，保证角色、地块、
  背景在同一世界里，而不是各画各的。
- **构图预留功能区**：单位立绘按可扣底的透明轮廓产出（便于叠放与换形态）；场景/战斗背景
  全幅铺满，并**预留承载 UI 与单位的地面/前景空区**，避免细节与血条、指令、飘字打架。
- **经皮肤层进入，不硬编码**：每个资产对应一个视觉键，缺图回退灰盒；美术因此可与玩法
  并行推进、随时替换，不改动引擎。
